多媒体取证 Agent 学习笔记：从项目骨架到 PDF 上传解析

发表于2026-05-31|更新于2026-05-31|AI
|阅读量:

今天继续推进 Multimedia-Forensics-Agent 项目。

这个项目的定位是：面向水印、隐写、图像取证论文的智能研读与实验辅助 Agent。它不是普通聊天机器人，而是希望围绕科研阅读和实验辅助流程，逐步实现论文上传、文本解析、RAG 问答、论文结构化分析、图像指标计算和 Markdown 报告生成。

今天主要完成的是项目的前两步：先搭好前后端骨架，再跑通 PDF 上传与文本解析链路。

1. 项目当前目标

这个项目后续希望支持：

  • 上传论文 PDF 并提取文本
  • 基于论文内容进行 RAG 问答
  • 提取论文的研究问题、方法、数据集、指标和实验结论
  • 对多篇论文进行横向对比
  • 计算图像质量指标，例如 MSE、PSNR、SSIM
  • 生成 Markdown 实验分析报告

目前还没有进入完整 RAG 和 Agent 阶段，今天的重点是先把“PDF 进入系统并变成可处理文本”这条最小链路跑通。

8. 今天踩到的问题

1. 中文注释编码问题

代码和博客里都出现了中文乱码。

这说明后面需要统一使用 UTF-8 编码保存文件。否则中文标题、注释和博客内容都会变成乱码，影响阅读和维护。

2. 注释可能吞掉代码

Python 里 # 后面的内容都会被当成注释。

如果把真正代码写在 # 后面，代码就不会执行。今天在 import 和 PDF 文本提取时都需要特别注意这个问题。

3. pages_preview 不应该返回全文

一开始容易直接返回所有页面文本，但这样对长论文很浪费资源。

更合理的做法是只返回前几页，并且每页只截取前几百个字符作为预览：

这样前端展示更快，也不会把响应体变得太大。

10. 下一步计划

下一步准备进入 RAG 的基础部分：

  1. 实现 text_splitter.py，把论文文本切成 chunks
  2. 实现一个简单检索模块，先用轻量方式跑通，不急着上复杂向量数据库
  3. 增加 /api/papers/ask 接口
  4. 在 Streamlit 前端加入论文问答输入框
  5. 返回相关 chunk 和答案来源页码

等这条链路跑通后，这个项目才会真正从“PDF 解析工具”进入“论文研读 Agent”的阶段。
